# Release Notes — v0.2.0

**Release date:** 2024-XX-XX

---

## Highlights

Phase 2 & 3 of the Thunderbird SDK: full hardware integration, production
logging, ABI stability, diagnostics, and a complete documentation suite.

---

## New Features

- **Hardware Transport Layer** — Ethernet (`eth://`) and USB (`usb://`)
  connectivity with automatic reconnection and heartbeat monitoring.
- **VLP-16 Packet Parser** — Decode Velodyne VLP-16 packets into
  `LidarFrame` with spherical→Cartesian conversion.
- **LiDAR Frame Assembler** — Accumulate packets into complete 360°
  frames with gap and drop detection.
- **Device Health Monitor** — Real-time health scoring (0.0–1.0) with
  rate monitoring, CRC checking, and connection flap detection.
- **Clock Service** — OLS-based host↔hardware clock drift estimation
  with outlier rejection.
- **TimeSyncEngine** — Cross-sensor alignment producing `SyncedFrame`
  bundles (LiDAR + IMU[] + Camera).
- **spdlog Logging System** — Per-module loggers with compile-time level
  stripping and rotating file output.
- **Diagnostics Manager** — Background metrics collection (1 Hz) with
  JSON export and custom collector registration.
- **ABI Stability** — Inline ABI namespace, `THUNDERBIRD_API` visibility
  macros, and compile-time version validation.

---

## Improvements

- `DeviceManager` now exposes `diagnostics_json()` for one-call metrics.
- Perception tracker uses Mahalanobis distance for association (was IoU).

---

## Breaking Changes

None — v0.2.0 is the first release with a public ABI contract.

---

## Known Issues

- GPU perception backends (`GpuPointPillars`, `GpuCenterPoint`) require
  manual CUDA toolkit installation.
- macOS builds do not support USB transport (Ethernet only).
